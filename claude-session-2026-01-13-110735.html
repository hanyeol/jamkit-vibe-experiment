   259â†’```sbss

   270â†’### Property Specifications
   271â†’
   272â†’```ebnf
   273â†’PropertySpec    ::= ListSpec | BlockSpec
   274â†’
   275â†’ListSpec        ::= ":" PropertyList
   276â†’
   277â†’BlockSpec       ::= "{" PropertyBlock "}"
   278â†’
   279â†’PropertyList    ::= Property {"," Property}
   280â†’
   281â†’PropertyBlock   ::= {PropertyLine}
   282â†’
   283â†’PropertyLine    ::= PropertyKey ":" PropertyValue ";"
   284â†’
   285â†’PropertyKey     ::= Identifier
   286â†’
   287â†’PropertyValue   ::= QuotedString | UnquotedValue

   291â†’
   292â†’UnquotedValue   ::= {ValueChar}
   293â†’
   294â†’StringChar      ::= /* Any character except quote or backslash */

   298â†’
   299â†’ValueChar       ::= /* Any character except whitespace, comma, and semicolon */

   302â†’**List Syntax Examples:**
   303â†’```sbss
   304â†’@root: width=100%, height=100%, background=#fff
   305â†’

   310â†’```sbss

   324â†’### Variable Resolution in Properties
   325â†’
   326â†’Variables defined with `$` can be used in property values:
   327â†’
   328â†’```sbss

   352â†’### Scope and Naming
   353â†’
   354â†’#### Global Scope
   355â†’
   356â†’When SBSS is parsed independently (not from within an SBML file):
   357â†’- Props and styles have global scope
   358â†’- Names are used as-is
   359â†’
   360â†’**Example (global.sbss):**
   361â†’```sbss
   362â†’@my-props: padding=10dp
   363â†’#my-style: color=#ff0000
   364â†’```
   365â†’
   366â†’These become `my-props` and `my-style` globally.
   367â†’
   368â†’#### Local Scope
   369â†’
   370â†’When SBSS is parsed from within an SBML file (via `=import`):
   371â†’- Props and styles are scoped to the SBML file
   372â†’- Names are prefixed with `sbml:<filename>:`
   373â†’
   374â†’**Example (in document.sbml):**
   375â†’```sbml
   376â†’=import "styles.sbss"
   377â†’```
   378â†’
   379â†’If `styles.sbss` contains:
   380â†’```sbss
   381â†’@my-props: padding=10dp
   382â†’#my-style: color=#ff0000
   383â†’```
   384â†’
   385â†’They become `sbml:document.sbml:my-props` and `sbml:document.sbml:my-style`.
   386â†’
   387â†’Names starting with `sbml:` are always global and not prefixed.
   388â†’
   389â†’### Line Continuation
   390â†’
   391â†’Lines ending with `\` are continued on the next line:
   392â†’
   393â†’```sbss
   394â†’$LONG_URL = "https://example.com/very/long/path/\
   395â†’that/continues/on/next/line"
   396â†’

   398â†’       height=100%, \
   399â†’       background=#ffffff
   400â†’```
   401â†’
   402â†’### Comments
   403â†’
   404â†’SBSS does not have explicit comment syntax. However, you can use conditionals that always evaluate to false:
   405â†’
   406â†’```sbss
   407â†’if 1 == 0
   408â†’    This is effectively a comment
   409â†’    @root: this-will-never-apply=true
   410â†’end
   411â†’```
   412â†’
   413â†’Or use undefined variables (which will log warnings but not cause errors):
   414â†’
   415â†’```sbss
   416â†’$_COMMENT = "This is documentation but will show warnings"
   417â†’```
   418â†’
   419â†’### Lexical Elements
   420â†’
   421â†’```ebnf
   422â†’Identifier      ::= Letter {Letter | Digit | "-" | "_" | ":"}
   423â†’
   424â†’Letter          ::= "a".."z" | "A".."Z"
   425â†’
   426â†’Digit           ::= "0".."9"

   435â†’### Special Considerations
   436â†’
   437â†’#### Import Restrictions
   438â†’
   439â†’- Only `.sbss` files can be imported (enforced by parser)
   440â†’- Recursive imports are detected and cause parsing errors
   441â†’- Imports are relative to the current file's directory
   442â†’
   443â†’#### Variable Naming
   444â†’
   445â†’- Variable names must start with `$`
   446â†’- By convention, private/internal variables start with `$_`
   447â†’- Variables are case-sensitive
   448â†’
   449â†’**Examples:**
   450â†’```sbss
   451â†’$PUBLIC_VAR = value
   452â†’$_private_var = value
   453â†’$camelCase = value
   454â†’$snake_case = value
   455â†’```
   456â†’
   457â†’#### Property Block Parsing
   458â†’
   459â†’When a block is opened with `{`, the parser collects all lines until the closing `}`:
   460â†’
   461â†’```sbss
   462â†’@root {
   463â†’    width: 100%;
   464â†’    height: 100%;
   465â†’}
   466â†’```
   467â†’
   468â†’The block content (between `{` and `}`) is then parsed as a property list with:
   469â†’- `:` as key-value separator
   470â†’- `;` as property separator
   471â†’- Whitespace is trimmed
   472â†’
   473â†’## Evaluation Order
   474â†’
   475â†’1. **Variable definitions** are evaluated first
   476â†’2. **Conditionals** are evaluated based on current variable state
   477â†’3. **Style definitions** are collected and stored
   478â†’4. When SBML is parsed, styles are applied based on:
   479â†’   - Root properties (`@root`)
   480â†’   - Tag-based properties (`%tag`)
   481â†’   - Path-based properties (`/path/to/section`)
   482â†’
   483â†’## UTF-8 Support
   484â†’
   485â†’- SBSS files must be encoded in UTF-8
   486â†’- Optional UTF-8 BOM (0xEF 0xBB 0xBF) at the beginning is automatically skipped
   487â†’- All identifiers and string values support Unicode characters
   488â†’
   489â†’## Error Handling
   490â†’
   491â†’The parser handles errors gracefully:
   492â†’- Undefined variables log warnings but don't stop parsing
   493â†’- Syntax errors in property lists log warnings
